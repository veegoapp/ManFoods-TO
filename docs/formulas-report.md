## ١. مقدمة — كيف يعمل الفلتر

### الفلاتر المتاحة في معظم الصفحات

| الفلتر | معناه |
|--------|-------|
| `month` / `year` | الشهر والسنة النهاية للفترة |
| `fromMonth` / `fromYear` | بداية فترة متصلة |
| `months` | شهور محددة مفصولة بفاصلة مثل `1,3,5` |
| `store` | فرع معين |
| `om` | مدير عمليات معين |
| `oc` | مستشار عمليات معين |

### قاعدة تحديد الفترة الزمنية

**لما تكتب `months` مع `year`:**
النظام بياخد الشهور دي بالضبط في السنة دي — مش لازم تكون متتالية.

**لما تكتب `fromMonth` و `fromYear` و `month` و `year`:**
النظام بيوسع كل الشهور بين البداية والنهاية.

**لو ماكتبتش حاجة:**
النظام بياخد تلقائياً آخر شهر موجود في قاعدة البيانات.

## ٢. صفحة الـ Dashboard — معدل الـ Turnover الرئيسي

### المعادلة

```
معدل الـ Turnover = ( مجموع الاستقالات ÷ متوسط عدد الموظفين ) × 100
```

### طريقة الحساب

**متوسط عدد الموظفين:**
النظام بياخد عدد الموظفين من كل شهر في الفترة المحددة ويحسب المتوسط.

مثال على 3 شهور:

```
متوسط الموظفين = ( عدد يناير + عدد فبراير + عدد مارس ) ÷ 3
```

**مجموع الاستقالات:**
جمع كل الاستقالات في الشهور المحددة.

**نتيجة معادلة الـ Turnover:**

```
معدل الـ Turnover = ( مجموع الاستقالات ÷ متوسط الموظفين ) × 100
```

النتيجة مقربة لـ خانتين عشريتين.

### الحالة الافتراضية

لو ماحددتش أي فلتر — النظام بياخد آخر شهر موجود في جدول الموظفين تلقائياً.

### الفلاتر المتاحة

فرع — مدير عمليات — مستشار عمليات — فترة زمنية.

## ٣. مقارنة الفروع

### المعادلة

```
نسبة Turnover للفرع = ( استقالات الفرع ÷ عدد موظفي الفرع ) × 100
```

النتيجة مقربة لـ خانة عشرية واحدة.

### مصدر البيانات

- **عدد الموظفين:** بياخده من آخر شهر في الفترة المحددة فقط (الـ anchor).
- **الاستقالات:** بيجمعها من كل الشهور المحددة.

### ملاحظة مهمة

لو اخترت 3 شهور مثلاً — الاستقالات هتكون مجموع الـ 3 شهور، لكن عدد الموظفين هو عدد الشهر الأخير بس. ده طبيعي لأن الـ headcount هو صورة لحظية.

### ترتيب النتائج

الفروع بتتعرض مرتبة تنازلياً من أعلى نسبة Turnover لأقلها.

## ٤. Smart Insights

بتعتمد على نتائج مقارنة الفروع وبتولد ملاحظات تلقائية.

### أ. أعلى فرع في الـ Turnover

بياخد أول فرع في القايمة المرتبة تنازلياً.

### ب. أفضل فرع

بياخد آخر فرع في القايمة (أقل نسبة) بشرط أن يكون عنده موظفين.

### ج. كشف القفزات المفاجئة (Spikes)

```
الفرق = نسبة الفرع هذه الفترة - نسبة الفرع الفترة السابقة
```

لو الفرق أكبر من أو يساوي 5% — يعتبره قفزة مفاجئة.
بيعرض أعلى 3 فروع فيهم قفزة.

### طريقة تحديد الفترة السابقة للمقارنة

**لو اخترت شهر واحد:**
الفترة السابقة = الشهر اللي قبله مباشرة.

**لو اخترت 3 شهور متتالية مثلاً يناير→مارس:**
الفترة السابقة = أكتوبر→ديسمبر (نفس العدد بالضبط).

**لو اخترت شهور محددة مثلاً 1,3,5 من 2024:**
الفترة السابقة = نفس الشهور 1,3,5 من 2023.

### د. الاتجاه العام

```
الفرق = مجموع استقالات هذه الفترة - مجموع استقالات الفترة السابقة
```

- لو الفرق موجب = **Worsening** (الأوضاع بتسوء)
- لو الفرق سالب = **Improving** (الأوضاع بتتحسن)
- لو الفرق صفر = **Stable** (مستقر)

### هـ. أسوأ مستشار عمليات (OC)

```
نسبة الـ OC = ( مجموع استقالات كل فروعه ÷ مجموع موظفي كل فروعه ) × 100
```

ده متوسط موزون — مش مجرد متوسط النسب — عشان الفروع الكبيرة تاخد وزن أكبر.

### و. أسوأ مدير عمليات (OM)

نفس معادلة الـ OC بالضبط.

## ٥. تحليل الـ OC والـ OM

نفس المعادلة الموزونة اللي في الـ Smart Insights، لكن بتعرض كل المسؤولين مش أسوأ واحد بس.

```
نسبة المسؤول = ( مجموع استقاالاته في كل فروعه ÷ مجموع موظفيه في كل فروعه ) × 100
```

مرتبة تنازلياً من الأعلى للأقل.

## ٦. صفحة الـ Retention — معدل الاحتفاظ

### المفهوم الأساسي

الـ Retention بيشتغل بنظام **الدفعات (Cohorts)** — كل دفعة هي مجموعة موظفين اتعيّنوا في شهر معين.

### المعادلة

```
معدل الاحتفاظ = ( عدد الموظفين اللي فضلوا ÷ إجمالي الدفعة ) × 100
```

النتيجة مقربة لـ خانة عشرية واحدة.

### نقاط القياس (Milestones)

| النقطة | عدد الأيام |
|--------|-----------|
| 6 أشهر | 182 يوم |
| سنة | 365 يوم |
| سنتين | 730 يوم |
| 3 سنوات | 1095 يوم |
| 4 سنوات | 1460 يوم |
| 5 سنوات | 1825 يوم |

### شرط مهم — CohortReaches

الدفعة بتتحسب في نقطة قياس **بس لو** انتهت الفترة الكافية من وقت تعيينها.

مثال: دفعة يناير 2024 مش بتتحسب في نقطة "سنة" غير بعد يناير 2025.

### طريقة حساب وقت بقاء الموظف

```
أيام الخدمة = تاريخ الاستقالة - تاريخ التعيين
```

لو الموظف لسه شغال = أيام الخدمة = null (يعني فضل)

### الفلاتر المتاحة

فرع — مدير عمليات — مستشار عمليات — فترة زمنية للدفعات (من أي تاريخ تعيين لأي تاريخ).

### الحالة الافتراضية

لو ماحددتش فترة — بياخد كل الدفعات المتاحة في قاعدة البيانات.

---

## ٧. منحنى البقاء (Survival Curve)

نفس منطق الـ Milestones بالضبط لكن بنقاط أكتر.

### نقاط المنحنى

| النقطة | الوقت |
|--------|-------|
| Day 0 | يوم التعيين |
| 1 شهر | 30 يوم |
| 3 أشهر | 90 يوم |
| 6 أشهر | 182 يوم |
| سنة | 365 يوم |
| سنة ونص | 545 يوم |
| سنتين | 730 يوم |
| 3 سنوات | 1095 يوم |
| 4 سنوات | 1460 يوم |
| 5 سنوات | 1825 يوم |

### المعادلة

```
نسبة البقاء عند النقطة X = ( عدد اللي فضلوا أكتر من X يوم ÷ عدد الدفعة الكاملة ) × 100
```

نفس شرط الـ CohortReaches بيطبق هنا.

## ٨. ترتيب الفروع في الـ Retention

### المعادلة

```
معدل احتفاظ الفرع = ( عدد اللي فضلوا أكتر من 365 يوم ÷ عدد الدفعة ) × 100
```

بيشمل فقط الدفعات اللي عمرها تجاوز سنة كاملة.
مرتبة تنازلياً من الأعلى للأقل.

## ٩. توزيع سنوات الخدمة (Tenure Distribution)

### المفهوم

سؤاله: "الموظفين اللي شغالين دلوقتي — قديهم مشغولين؟"

### المعادلة

```
أيام الخدمة = آخر يوم في الشهر المحدد - تاريخ تعيين الموظف
```

بعدين بيصنف كل موظف في فئة:

| الفئة | الأيام |
|-------|--------|
| أقل من 6 أشهر | 0 → 181 يوم |
| 6 إلى 12 شهر | 182 → 364 يوم |
| سنة إلى سنتين | 365 → 729 يوم |
| سنتين إلى 3 | 730 → 1094 يوم |
| 3 إلى 4 سنوات | 1095 → 1459 يوم |
| 4 إلى 5 سنوات | 1460 → 1824 يوم |
| أكتر من 5 سنوات | 1825 يوم فأكتر |

### الحالة الافتراضية

لو ماحددتش شهر — بياخد تلقائياً آخر شهر موجود في قاعدة البيانات.

### لو حددت شهر

بيعرض صورة الموظفين كما كانوا في ذلك الشهر بالضبط.

### ملاحظة

ده مش متأثر باختيار أكتر من شهر — هو دايماً صورة لحظية عند شهر واحد فقط.

## ١٠. صفحة 90 يوم

### المفهوم

بيقيس نسبة الموظفين اللي غادروا قبل ما يكملوا 90 يوم من التعيين.

### المعادلة

```
نسبة الـ 90 يوم = ( عدد اللي استقالوا في أول 90 يوم ÷ إجمالي المعيّنين في الفترة ) × 100
```

### مصدر البيانات

بيشوف تاريخ التعيين لكل موظف — لو استقال وعدد أيام خدمته 90 يوم أو أقل يحسبه في البسط.

### علامة Provisional

لو الدفعة أُقفلت من أقل من 90 يوم — النتيجة بتتعلم بعلامة "مبدئية" يعني ممكن تتغير لحد ما تكمل الـ 90 يوم.

### مثال

```
الدفعة اتعيّنت يناير 2025
90 يوم = أبريل 2025
لو دلوقتي مارس 2025 — النتيجة Provisional
لو دلوقتي مايو 2025 — النتيجة نهائية
```

### الحالة الافتراضية للـ Trend Chart

بيعرض كل الدفعات المتاحة في قاعدة البيانات من أقدم تاريخ لأحدثه.

## ١١. صفحة الـ Scorecard

بتعرض أداء كل قائد أو مسؤول في 4 مقاييس.

### المقياس الأول — معدل الـ Turnover

```
معدل الـ Turnover = ( مجموع الاستقالات ÷ متوسط عدد الموظفين ) × 100
```

الحساب بياخد الشهور اللي كان المسؤول فيها بالفعل مسؤولاً عن الفرع ده.

### المقياس الثاني — نسبة المغادرة المبكرة (90 يوم)

```
نسبة المغادرة = ( عدد اللي استقالوا في أول 90 يوم ÷ إجمالي المعيّنين ) × 100
```

### المقياس الثالث — معدل البقاء 180 يوم

```
معدل البقاء = ( عدد اللي فضلوا أكتر من 180 يوم أو لسه شغالين ÷ إجمالي المعيّنين ) × 100
```

### ملاحظة مهمة على المقياسين 2 و 3

لما تحدد فترة زمنية — المقياسين بيتحسبوا على الموظفين اللي **اتعيّنوا خلال الفترة دي بالضبط**.
لو ماحددتش فترة — بيستخدم كل التاريخ المتاح.

### المقياس الرابع — نسبة الرضا في مقابلات الخروج

```
نسبة الرضا = ( عدد الإجابات الإيجابية ÷ إجمالي الإجابات ) × 100
```

### أبعاد الـ Scorecard

- `leader` — قائد الفرع
- `oc` — مستشار العمليات
- `om` — مدير العمليات

### الفلاتر المتاحة

أبعاد — مدير عمليات — مستشار عمليات — فترة زمنية (شهر أو شهور).

## ١٢. نظام الإنذار المبكر (Early Warning)

### المفهوم

بيحدد الموظفين الشغالين اللي عندهم مخاطر مغادرة مرتفعة — بناءً على مقارنتهم بالتاريخ الكامل للشركة.

### آلية التسجيل

كل موظف بياخد نقاط من 6 عوامل — كل عامل بيدي 1 إلى 3 نقاط. مجموع النقاط بيتحول لنجوم:

| مجموع النقاط | النجوم |
|-------------|--------|
| 0 → 2 | نجمة واحدة |
| 3 → 4 | نجمتان |
| 5 → 6 | 3 نجوم |
| 7 → 9 | 4 نجوم |
| 10 فأكتر | 5 نجوم |

كل ما النجوم أكتر — الخطر أعلى.

---

### العامل الأول — داخل نافذة الموظف الجديد

```
نسبة التقدم = أيام الخدمة ÷ حجم النافذة
```

**حجم النافذة** بيتحدد لكل وظيفة على حدة بناءً على متوسط وقت المغادرة التاريخي لأصحاب نفس الوظيفة.

القيم المتاحة بين 60 و 180 يوم — لو ماكانش فيه بيانات كافية يستخدم 90 يوم كافتراضي.

| نسبة التقدم | النقاط |
|------------|--------|
| أقل من 40% | نقطة واحدة |
| 40% إلى 74% | نقطتان |
| 75% فأكتر | 3 نقاط |

---

### العامل الثاني — نسبة مغادرة الفرع

```
نسبة مغادرة الفرع = ( عدد اللي غادروا مبكراً من الفرع ده ÷ إجمالي المعيّنين في الفرع ) × 100
```

**المقياس:** الانحراف المعياري — كيف نسبة الفرع ده مقارنة بمتوسط كل الفروع.

```
الحد الأول = متوسط الفروع + ( 1.0 × الانحراف المعياري )  → نقطة واحدة
الحد الثاني = متوسط الفروع + ( 1.5 × الانحراف المعياري ) → نقطتان
الحد الثالث = متوسط الفروع + ( 2.0 × الانحراف المعياري ) → 3 نقاط
```

**ملاحظة:** الانحراف المعياري الأدنى المعتمد هو 5% عشان يمنع التضخيم في البيانات القليلة.

---

### العامل الثالث — نسبة مغادرة الوظيفة

نفس منطق العامل الثاني لكن بيقارن الموظف بزملائه في نفس المسمى الوظيفي.

---

### العامل الرابع — نسبة مغادرة الجنس

نفس المنطق لكن بيقارن بحسب الجنس.

---

### العامل الخامس — سمعة الفرع في مقابلات الخروج

```
نسبة الاستياء = ( عدد الإجابات السلبية ÷ إجمالي الإجابات ) × 100
```

**الإجابات السلبية** = كل إجابة فيها "لا أوافق" أو "ضعيفة" أو "لا".

الفرع بيدخل في الحساب بس لو عنده 3 مقابلات خروج على الأقل.

نفس منطق الانحراف المعياري بيطبق عليه.

---

### العامل السادس — سجل قائد الفرع

```
نسبة المغادرة المبكرة للقائد = ( مجموع المغادرين مبكراً من كل فروعه ÷ مجموع موظفيه في كل فروعه ) × 100
```

ده متوسط موزون لكل الفروع اللي أشرف عليها القائد ده في أي وقت.

نفس منطق الانحراف المعياري بيطبق.

---

### نقطة مهمة في نافذة الموظف الجديد — Peak Window

بعد ما الموظف يعدي نافذته الأساسية، النظام بيدور على "النافذة الثانية" وهي الفترة اللي فيها ناس كتير استقالت تاريخياً بعد الفترة الأولى.

بيحسبها بقسمة التواريخ لفئات عرضها 30 يوم ويلاقي الفئة الأكتر تكراراً.

---

### الحالة الافتراضية

بيشوف الموظفين الشغالين في **آخر شهر متاح** — أو الشهر المحدد لو اخترت.

## ١٣. مقابلات الخروج (Exit Interviews)

### نسبة الرضا

```
نسبة الرضا = ( عدد الإجابات الإيجابية ÷ إجمالي الإجابات ) × 100
```

### نسبة الاستياء

```
نسبة الاستياء = ( عدد الإجابات السلبية ÷ إجمالي الإجابات ) × 100
```

### تصنيف الإجابات

| الإجابة | التصنيف |
|---------|---------|
| أوافق / جيدة / نعم / عالية / كبيرة | إيجابي |
| لا أوافق / ضعيفة / لا | سلبي |
| محايد / لا أوافق ولا أعارض / مقبولة | محايد |

### الأسئلة المستخدمة في حساب صحة الفرع

- أعباء العمل والظروف
- العدالة في المعاملة
- التواصل مع الإدارة
- التدريب والتطوير
- معالجة الشكاوى

## ١٤. تكلفة الـ AI

### المعادلة

```
التكلفة = ( عدد رموز الإدخال ÷ 1,000,000 × 0.10 دولار ) + ( عدد رموز الإخراج ÷ 1,000,000 × 0.40 دولار )
```

| النوع | السعر |
|-------|-------|
| رموز الإدخال (Prompt) | 0.10 دولار لكل مليون رمز |
| رموز الإخراج (Completion) | 0.40 دولار لكل مليون رمز |
